## Context

当前 `SideDrawer` 位于 `src/features/figma/components/shared.tsx`，承担全站主要导航与钱包快捷操作入口，包括菜单切换、关闭抽屉、钱包连接、切换钱包和断开连接等核心交互；但其视觉依然停留在旧版深色黑金体系，与已经完成升级的首页、认购中心和社区页不一致。  
由于前面几个页面已经建立了新的浅色金白视觉语言，这次更适合继续复用已验证过的共享风格，只对抽屉容器、头部、菜单项与钱包区做展示层重构，而不是调整交互结构。

## Goals / Non-Goals

**Goals:**
- 让左侧菜单栏与新版首页、认购中心、社区页形成统一的品牌视觉
- 保留现有菜单导航与钱包相关交互行为
- 提升抽屉头部、菜单项选中态、钱包状态区和操作按钮的层次感与精致度
- 尽量复用现有白卡、浅金强调、黑金 CTA、边框和阴影模式

**Non-Goals:**
- 不改动菜单数据源 `menuItems`
- 不改动钱包连接、切换、断开逻辑和相关回调协议
- 不新增依赖，不引入新的抽屉组件库
- 不在本次变更中扩展新菜单功能或新的钱包业务能力

## Decisions

### 1. 保留抽屉结构与交互，只升级视觉表达
- 决策：继续沿用 `SideDrawer` 当前的整体结构、菜单项循环、钱包状态分支和按钮回调，只替换背景、容器、标签、选中态和按钮样式。
- 原因：当前抽屉逻辑成熟稳定，本次核心目标是统一视觉。
- 备选方案：重做抽屉结构或拆成多个子组件。收益有限，且增加回归成本。

### 2. 抽屉改为浅色金白主视觉，但保留品牌强调
- 决策：将当前深色抽屉底替换为浅色面板、白色卡片和柔和金色强调，同时保留少量黑金 CTA 用于关键钱包操作。
- 原因：这样既能和主页面统一，也能维持品牌识别度。
- 备选方案：继续保留深色抽屉，只调整局部细节。这样仍会与新版主页面割裂。

### 3. 菜单区和钱包区采用分层卡片式布局
- 决策：把头部品牌区、钱包状态区、菜单导航区做成清晰分段，通过轻边框、白卡、选中态高亮和行内图标提升扫读效率。
- 原因：左侧菜单栏承担导航和状态展示双重职责，需要更清晰的信息层次。
- 备选方案：继续使用纯列表铺排。这样视觉品质和信息分组都不够强。

### 4. 菜单选中态强调“当前所在页面”
- 决策：为激活菜单项提供更明显但统一的高品质高亮，包括图标底、文字颜色、边框/背景和右箭头状态。
- 原因：抽屉的主要任务之一是帮助用户确认当前位置与快速切换。
- 备选方案：仅保留旧版文字高亮。这样反馈不够明显。

## Risks / Trade-offs

- [浅色抽屉与页面背景过近导致边界不明显] → 用面板阴影、右侧分隔和卡片层次拉开
- [钱包按钮全部浅色化后主次关系不清] → 保留关键 CTA 的黑金强调，其余使用浅色次级按钮
- [菜单项选中态过重影响整体轻盈感] → 控制高亮饱和度，采用浅金底 + 深色文字而非重色块
- [共享组件被多页面复用，视觉调整易影响全局] → 实施时仅调整 `SideDrawer` 相关样式，避免波及其他共享组件

## Migration Plan

这是纯前端共享组件展示层升级，不涉及接口或数据迁移。上线方式为直接替换当前 `SideDrawer` 样式实现；如出现问题，可回滚本次前端改动恢复旧版抽屉样式。
